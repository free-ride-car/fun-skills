# Skill 模板

使用此模板创建新的 Claude Skill。

---

## SKILL.md 模板

```markdown
---
name: your-skill-name
description: 简短描述这个 skill 的功能。当用户请求相关任务时，Claude 会根据此描述判断是否激活该 skill。
metadata:
  author: xxx
  version: "1.0"
---

# Skill 名称

简要说明这个 Skill 的用途和功能。

## 使用场景

当用户请求以下内容时激活此 skill：
- "触发词1"、"触发词2"
- "相关请求描述"

## 参数配置

### 必要参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `param1` | 参数说明 | 默认值 |

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `param2` | 参数说明 | 默认值 |

## 执行流程

### 第一步：步骤名称

详细描述这一步需要做什么。

```bash
# 示例命令
your-command-here
```

### 第二步：步骤名称

继续描述后续步骤...

## 输出格式

描述最终输出的格式和结构。

```markdown
# 输出模板

内容结构...
```

## 示例对话

**用户**: 示例请求

**助手**: 示例响应
```

---

## 命名规范

- Skill 名称使用 kebab-case（小写字母，单词用连字符分隔）
- 文件夹名称与 skill name 保持一致
- 描述要简洁明了，便于 Claude 判断何时使用

## 最佳实践

1. **明确触发条件**：清晰定义何时应该使用这个 Skill
2. **详细执行流程**：步骤要足够详细，让 Claude 能够准确执行
3. **提供示例**：包含示例对话帮助理解使用方式
4. **定义输出格式**：明确输出的结构和格式要求
